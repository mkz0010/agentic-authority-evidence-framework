# Security Policy

AAEF is a documentation and control framework project.

## Reporting Unsafe Guidance or Security Concerns

If you believe a document, example, or control recommendation introduces unsafe guidance, please report it through one of the following channels:

1. **GitHub Security Advisory / Private Vulnerability Reporting** for this repository, if enabled.
2. **A public GitHub issue** only when the report does not include sensitive details.

Do not submit credentials, private keys, customer data, internal incident details, non-public architecture diagrams, or exploit details in public issues.

If you publish this repository under an organization or personal account, replace this section with a dedicated security contact email or enable GitHub private vulnerability reporting before broad public promotion.

## Scope

Security feedback may include:

- unsafe implementation examples,
- misleading control language,
- missing abuse cases,
- privacy risks in evidence examples,
- overcollection risks in logging,
- recommendations that could increase operational risk,
- ambiguous guidance that may lead to unsafe agent autonomy.

## Out of Scope

AAEF does not provide emergency incident response, legal advice, compliance certification, or vulnerability remediation services.
