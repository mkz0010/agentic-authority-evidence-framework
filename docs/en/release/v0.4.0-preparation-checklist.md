# v0.4.0 Release Preparation Checklist

## Status

This document is a release preparation checklist for the AAEF v0.4.0 Public Review Draft.

It is intended to help maintainers verify repository consistency before creating the v0.4.0 tag and GitHub Release.

## Release Goal

The goal of v0.4.0 is to expand the v0.3.x public review baseline with enterprise assessment usability guidance.

The v0.4.0 release includes:

- control catalog versioning and change impact guidance,
- measurable testing procedures and pass criteria,
- machine-readable testing procedure draft,
- High-Impact and Audit-Grade pre-qualification guidance,
- Trusted Control Boundary integrity requirements,
- external framework mapping methodology,
- initial conservative external framework mapping draft,
- validation for testing procedures and external mappings.

## Release Character

The v0.4.0 release is a public review draft.

It is not:

- a certification scheme,
- a formal standard,
- an implementation conformance claim,
- an audit opinion,
- compliance equivalence,
- or a claim of complete mitigation.

## Pre-Tag Checks

Before tagging v0.4.0, verify:

- [ ] README document status references v0.4.0.
- [ ] README citation references v0.4.0.
- [ ] CHANGELOG includes v0.4.0 release notes.
- [ ] One-page Overview current status references v0.4.0.
- [ ] Repository structure includes v0.4.0 documents and mapping artifacts.
- [ ] CITATION.cff references v0.4.0.
- [ ] Localized README files are updated or intentionally deferred.
- [ ] All validation scripts pass.

## Validation Commands

    python tools/validate_external_mappings.py
    python tools/validate_testing_procedures.py
    python tools/validate_assessment_profiles.py
    python tools/validate_assessment_worksheet.py
    python tools/validate_control_catalog.py
    python tools/validate_evidence_schema.py

## Tagging Commands

    git tag v0.4.0
    git push origin v0.4.0

Create the GitHub Release from tag `v0.4.0`.

## Post-Release Checks

After publishing the release, verify:

- [ ] Tag `v0.4.0` exists locally.
- [ ] Tag `v0.4.0` exists on GitHub.
- [ ] GitHub Release is marked as latest if appropriate.
- [ ] README renders correctly.
- [ ] Validation badge is visible.
- [ ] Open issues and milestones reflect the next planning stage.
